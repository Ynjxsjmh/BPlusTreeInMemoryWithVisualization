How to connection to the database server for hw1

The Postgres database server is setup in the edlab.  Here is necessary information for you to log on the edlab.

The relevant database connection info:

  • database server host name: db-edlab.cs.umass.edu  
  • port: 7445
  • username: the same as your spire/oit username
  • password: your student ID number
  • database name: the same as your spire/oit username
There are two ways to connect to the database in order to run the queries needed to complete homework 1. You can use:(1) psql from a terminal window. (2) a GUI called pgadmin. pgadmin is a nice tool and we recommend using it.  Both are explained below.

-psql

psql is a terminal-based front-end to PostgreSQL. It enables you type in queries interactively, issue them to PostgreSQL, and see the query results.

First, ssh to an edlab machine.

$ ssh user@elnux7.cs.umass.edu  (Note: elunx1~elnux7 are all ok, I will take elnux7 as an example.)

Second, run the psql command

elnux7> psql -h db-edlab -p 7445 -d <database name>

You are now ready to issue queries.

A complete list of psql instructions can be found here. Notably, \q allows you to quit psql.

-pgadmin

pgadmin is a nice GUI that allows you to connect the database server and run queries. Setting it up requires 3 steps.

  1. Download and install pgadmin from the official site (Download for Windows/Mac/other). The current version is 1.8.4.
  2. Set up an ssh tunnel. (more details are provided below)
  3. Connect to the database server with pgadmin. (details below)

Because the edlab machines restrict access to IPs within the edlab, an ssh tunnel must be setup to provide remote access to the database server from your non-edlab machine. The ssh tunnel can be setup as follows:

Using command to set up an SSH tunnel:

The ssh tunnel can be setup as follows where haopeng is the username:

$ ssh -L 7445:db-edlab.cs.umass.edu:7445 haopeng@elnux7.cs.umass.edu

The -L 7445:db-edlab.cs.umass.edu:7445 is in the form of -L local-port:host:remote-port.of the form.

After using any of the two methods above, you can connect to the database server with pgadmin. Open pgadmin and go to File -> Add Server ... Fill in the form as shown in the screenshot below (haopeng is my username):
